# Civic Engagement — Security & Code Quality Audit
**Date:** 2026-03-19
**Auditor:** Claude (vibe-security skill + manual review)
**Scope:** index.html, sw.js, manifest.json, scripts/, data/

---

## CRITICAL

### C1. XSS in Service Cards — Unescaped API Data in innerHTML
**File:** `index.html` lines 3122-3123
```js
<div class="service-name">${name}</div>
<div class="service-address">${addr}</div>
```
`name` and `addr` come from the Geoapify API response and are inserted into innerHTML without `escHtml()`. A malicious or compromised Geoapify response could inject arbitrary HTML/JS.
**Fix:** Wrap both in `escHtml()`: `${escHtml(name)}`, `${escHtml(addr)}`.
**Status:** ✅ Fixed 2026-03-19

### C2. XSS in Rep Cards — JSON.stringify Injected into onclick Attribute
**File:** `index.html` lines 3027, 3038
```js
onclick='openRepProfile(${JSON.stringify(rep).replace(/'/g, "&#39;")})'
onclick='event.stopPropagation();openModal(${JSON.stringify(rep).replace(/'/g, "&#39;")})'
```
Only single quotes are escaped. `JSON.stringify` does not escape `<`, `>`, or `&` — these can terminate the surrounding HTML. A malformed response from the Represent API could break out of the attribute context.
**Fix:** Use `data-*` attributes + addEventListener, like `buildLeaderCard()` already does correctly.
**Status:** ✅ Fixed 2026-03-19

### C3. XSS in Budget Bubbles — AI Response Inserted Raw into innerHTML
**File:** `index.html` lines 4267-4281
`parsed.tldr` and `parsed.breakdown` (from Claude API / fallback data) inserted raw into innerHTML.
**Fix:** Apply `escHtml()` to `parsed.tldr`, `parsed.breakdown`, and `escAttr()` to `sourceUrl`. Fix before re-enabling budget tab.
**Status:** ✅ Fixed 2026-03-19

## HIGH

### H1. No SRI Hashes on External CDN Scripts
**File:** `index.html` lines 42-43
Leaflet CSS and JS loaded from cdnjs.cloudflare.com without `integrity` or `crossorigin` attributes.
**Fix:** Add SRI hashes to both Leaflet tags.
**Status:** ✅ Fixed 2026-03-19

### H2. CSP Allows `unsafe-inline` for Scripts
**File:** `index.html` line 9
`script-src 'self' 'unsafe-inline'` weakens XSS protection. Unavoidable with fully inline JS architecture.
**Fix:** Long-term: move JS to external file + nonce CSP. Short-term: fixing XSS issues above is the priority.
**Status:** 🔲 Deferred (requires full architecture change)

### H3. Budget Card Data Inserted Without Escaping
**File:** `index.html` lines 4173-4185
`cat.icon`, `cat.label`, `cat.amount` inserted raw (safe since hardcoded, but fragile).
**Fix:** Apply `escHtml()` to all three values.
**Status:** ✅ Fixed 2026-03-19

## MEDIUM

### M1. Service Worker Cache-First, No Revalidation
**File:** `sw.js` line 38
Cache-first with no revalidation means users can be stuck on stale versions indefinitely.
**Fix:** Use stale-while-revalidate pattern.
**Status:** ✅ Fixed 2026-03-19

### M2. Duplicate Event Listener on Modal Backdrop
**File:** `index.html` lines 3598-3600 and 4354-4356
Identical listener registered twice — fires handler twice.
**Fix:** Remove duplicate at line 4354.
**Status:** ✅ Fixed 2026-03-19

### M3. No Input Validation on Address
**File:** `index.html` line 2840
Address sent to Geoapify with only `.trim()` — no length limit or character validation.
**Fix:** Add max length (200 chars) and strip non-printable characters.
**Status:** ✅ Fixed 2026-03-19

### M4. SSL Verification Disabled in Scraper
**File:** `scripts/scrape_burlington.py` line 47
`SSL_VERIFY = False` + `urllib3.disable_warnings()` — allows MITM against eSCRIBE portal.
**Fix:** Use certifi with proper cert bundle.
**Status:** ✅ Fixed 2026-03-19

### M5. Address Stored in URL Query Parameter
**File:** `index.html` line 2842
`?q=` param puts user address in browser history/server logs, contradicting "addresses never stored" privacy claim.
**Fix:** Noted. Acceptable UX trade-off for shareable links — add clarification to privacy notice.
**Status:** 🔲 Acknowledged (by design for shareability)

### M6. Large Inline Budget Data
**File:** `index.html` lines 3606-3963
~360 lines of CITY_BUDGETS JSON inline while tab is disabled.
**Fix:** Move to `data/budgets.json`, fetch lazily when re-enabling.
**Status:** ✅ Fixed 2026-03-19

### M7. Google Fonts Privacy Leak
Google Fonts leaks visitor IPs to Google.
**Fix:** Self-host fonts (lower priority, no security risk).
**Status:** 🔲 Deferred

## LOW

### L1. Dead Code — BYOK Modal HTML Still in DOM
API key modal HTML still rendered even though JS is commented out.
**Fix:** Comment out or remove the modal HTML.
**Status:** ✅ Fixed 2026-03-19

### L2. No ARIA Roles or Focus Traps on Modals
No `role="dialog"`, `aria-modal`, `aria-labelledby`, or focus traps on any modal.
**Fix:** Add ARIA attributes and focus traps to all modals.
**Status:** ✅ Fixed 2026-03-19

### L3. `escHtml()` Doesn't Escape Quotes
Safe for text content but dangerous if accidentally used in attribute context.
**Fix:** Add `"` and `'` escaping as defense-in-depth.
**Status:** ✅ Fixed 2026-03-19

### L4. Theme Preference Not Persisted
Light/dark toggle resets on every visit. GrantMatch already does this correctly.
**Fix:** Read/write from localStorage on init.
**Status:** ✅ Fixed 2026-03-19

### L5. Missing ARIA Labels on Service Card Links
Service cards have no `aria-label`.
**Status:** ✅ Fixed 2026-03-19

### L6. Large `buildMeetingCard()` Function (~250 lines)
Hard to audit; consider splitting.
**Status:** 🔲 Deferred (low risk, refactor when re-enabling meetings tab)
